我为什么选择了 Markdown

因为一直要码字儿而且仅需要简单地排版的缘故，office word 那()的强大功能与不怎么美观的界面实在是不得我心. 于是就另寻他欢，找到了 Markdown. Markdown 只是一类语法，现在有很多的文本编辑器是专门在支持 Markdown 的基础上再进行功能的扩展.

我看中 Markdown 的另外一点是它可以支持代码的高亮与数学公式的插入，这对于自学编程的人来说是一大吸引.

使用了哪些 Markdown 编辑器

最开始使用的是 Typora，这个编辑器颜值与实力并存， 说它极简并不为过. 亮点在于『所见即所得』
然后找到了 Yu writer ，这个软件的细节处做的很好，戳中了我不少痛点. 而且只需要按照作者的说明修改一下便可以使用便携版. 但是 Yu writer 这个软件还处于 Beta 阶段，功能不完善(我觉得已经可以满足日常所需了，本身作为一款 Markdown 编辑器，不需要太多的功能)，用起来有一丢丢的卡. 比如删除大段文字`显示/隐藏文档库. 我把它当做本地的编辑器，看着还是十分养眼的

因为用过印象笔记(现已退坑)，就慕名找到了马克飞象，来个『双象合璧』. 但是马克飞象写下的笔记在印象笔记中只能看不能改，虽说是处于安全考虑吧，但还是点尴尬.
目前编辑文本使用的软件

前几天我在知乎，简书，CSDN 上看了几篇关于搭建个人博客的文章，就心血来潮做了一个. 平时自称有着『彪悍学习能力』的我顿时感到头秃，用了挺长的时间去搭建. 但是想想自己并不怎么以博客形式来发布自己写的东西，而且看着那个丑的一批的界面，连写作的欲望都没了. 就老老实实的在本地用 Typora 与 Yu writer .

Typora 因为对于数学公式与代码的强有力的支持，让我在总结 C 语言与数据结构的知识的时候挺方便.Yu Writer 则是用来写纯文本了，使用体验还是蛮愉快的

但是对于知识的总结，特别是很多知识的综合，已经不能从单方面的看怎么对它们进行方便地整理，我们更需要能够进行知识的关联与快速的检索. 比如可以进行笔记内联与添加标签，我使用的是为知笔记. 其实印象笔记与有道云笔记我都有用过.

只是为知笔记对于代码书写的支持好于另外两个. 而其他的方面我认为差异不大

怎么看日益增多的 Markdown 编辑器

目前使用 Mardown 进行写作的人已经不少了. 特别是在 Mac 平台上，有着很多比较有名的写作软件，几乎都支持 Markdown. Windows 平台嘛……我觉得 Office Word 就是最有名的写作软件，并且我们是可以安装插件来支持 Markdown 的. 其实我们需要的并不是众多的 Markdown 编辑器与各种黑科技，我们只是需要一个能让我们专注于写作这件事的工具. Mardown 的简易性符合了这一需求，我们可以发现很多的 Markdown 编辑器都有『专注模式』这个选项. 这也正提醒着使用者，不要过度地为文本的格式所干扰，专注于内容才是写作的真谛.

我们可能在为了一个比较好用的功能而使用着很多不同的 Markdown 编辑器. 比如我就是看中了能让我变 cool 的 Yu Writer 才使用它的(微笑)，当然啦，不排除自己本身很 Cool. 但不可为了使用而使用，否则就会出现面对屏幕干瞪眼，一上午憋不出来几个字儿的尴尬情况.

利器在手，如果不能善用，便会成为一个包袱.
