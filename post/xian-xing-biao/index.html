线性表

2019-11-30 Views 7 min read             繁
 

线性表是数据结构中为最基础和重要的一部分,后章的栈 队列 树等等都是在此基础上展开的

基本概念&比较

  1. 线性表的存储结构: 顺序存储结构&链式存储结构
  2. 两种存储结构特点的比较
顺序表 链表
分配方式 静态分配 动态分配
存储密度 1 小于1
存取方法 随机 顺序
插入删除的时间复杂度 O(n) O(1)
按序访问的时间复杂度 O(1) O(n)
存储密度: 一个节点中数组元素所占的存储单元与整个节点所占的存储单元之比

顺序存储的优点:

  • 方法简单,存储密度大
  • 随机访问,简洁便利

顺序存储的缺点:

  • 在做插入,删除操作时, 效率低
  • 静态分配,需要预先分配足够大的存储空间

链表的优点:

  • 动态分配,无需事先了解表的长度
  • 适合经常插入和删除的情况
  • 可以建立双向或循环链表提高效率

线性表的各种操作

结构体定义

  1. 顺序表的结构体定义
typedef struct{
    int data[maxsize];  //存放顺序表元素的数组
    int length;         //表的长度
}Sqlist;          //类型的定义
  1. 单链表节点定义
typedef struct LNode{
    int data;        //data存放节点数据域
    struct LNode *next;   //指向后继结点的指针
}LNode, *LinkList;

说明: 结点是内存中一片由用户分配的存储空间, 只有一个地址来表示它的存在, 没有显式的名称, 因此在分配链表空间的时候, 同时定义一个指针, 来存储这片空间的地址, 并且常用这个指针的名称来作为节点的名称.
LNode *A = (LNode*)malloc(sizeof(LNode));
含义是: 用户分配了一片LNode型空间, 即构造了一个LNode型的结点. 并定义一个名字为A的指针来指向这个结点, 同时将A当作这个结点的名字

顺序表的操作

初始化 给线性表相关参数赋值
求长度 求线性表的元素个数
取元素 取给定位置的元素值
定位 查找给定元素值的位置
插入 在制定位置插入给定的值
删除 删除指定位置的值
遍历 从头到尾扫描线性表
  1. 元素的插入（在L的第p个位置上插入新元素e）
void insertElem(Sqlist *L, int p, int e){
    int i;
    if(p<0 || p>L->length || L->length==maxsize)  //位置错误,或者表长已经达到最大值
        return 0;
    for(i=L->length-1; i>=p; --i)  //从后往前,逐个将元素往后移动一个位置
        L->data[i+1]=L->data[i];
    L->data[p]=e;  //将e插入到位置P
    ++(L->length);  //表长加一
    return 1;
}
  1. 元素的查找(查找第一个等于e的元素)
int findElem(sqlist *L , int e){
    int i;
    for(i=0; i<L->length; ++i)
        if(e==L->data[i])
            return i;  //若找到,则返回下标
    return -1;
}

void initList(sqlist *L){
    L->length = 0;
}
  1. 删除下表为P的元素
int deleteElem(sqlist *L, int p, int *e){
    int i;
    if(p<0 || p>L->length-1)
        return 0;
    e=L->data[p];  //将被删除的元素赋值给e
    for(i=p; i<L->length-1; i++)  //从p位置开始, 将后边的元素逐个向前移动一个位置
        L->data[i] = L->data[i+1];
    --(L->length);
    return 1;
}
  1. 求指定位置的算法(用e返回L中P位置上的元素)
int getElem(sqlist *L, int p, int *e){
    if(p<0 || p>L->length-1)
        return 0;
    e= L->data[p];
    return 1;
}

链表的操作（暂且讨论单链表）

  1. 初始化，构建头节点不存储数据的空表. 二级指针是为了解决malloc分配的内存在函数调用后失效的问题
void init_Linklist(LinkList *L){
    *L =(LinkList)malloc(sizeof(LNode));
    (*L)->next = NULL;
}
  1. 插入元素(头插法)
int insert_First(LinkList *L, ElemType x){
    LNode *p;
    p = (LNode *)malloc(sizeof(LNode));
    if(p==NULL)
        return OverFlow;
    p->data = x;  
    p->next = (*L)->next;  //p所指的新节点的指针域next指向L中开始的结点
    (*L)->next = p; //头结点的指针域next指向p结点,使得P成为新的开始结点
 }

  1. 删除指定元素
int delete_LinkList(LinkList *L, ElemType x){
    LNode *p, *q;
    p = *L;
    while(p->next != NULL){   //查找操作
        if(p->next->data == x)  //当匹配时就立刻结束
            break;
        p= p->next;  //不匹配时继续查找下一个
    }
    if(p->next == NULL)  //到达表尾
        return OK;
    else{       //删除操作
        q = p->next;
        p->next = q->next;
        free(q);
    }
}

通过习题得出的零碎知识点

在这一章节, 主要的习题时对于线性表两种结构的优缺点进行考查. 涉及到时间复杂度的计算和一些简单的操作题

  1. 常遇到的几种表的类型:

单链表; 单循环链表&各种衍生链表; 双链表&各种衍生链表

  1. 在其他情况相同时, 循环与非循环链表占用的内存空间是一样的
  2. 在链表中插入或删除一个结点, 需要修改该相邻结点的指针域. 如不特别指明, 通常只给出链表头结点的地址, 其他结点的地址只能从它的前驱或者后继得到.
  3. 在双向链表(单链表)中,进行插入操作时要注意: 从后往前进行操作, 例如 B插入到A C 之间. 则需要先将B C 之间的关系连接好, 再进行A B之间的关系连接. 否则会造成断裂的情况
  4. 对于非空的单循环链表L的特性:
  • 带有头结点的单循环链表没有空指针域
  • 当终端结点与头结点为同一个结点时, 即 P ==L , 链表为空
  • 当链表为空时, p指向头结点, 同时也是终端节点. 此时满足 p->next == L (注意L也代表的指向头结点的指针)
